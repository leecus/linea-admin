MINIMAL STYLE

极简后台组件库

这个 design style 面向高频后台系统：信息密度足够、视觉负担很低、状态表达克制， 用线、留白、字号和灰度建立秩序，而不是靠厚重卡片、阴影和强色块。

-- Quiet, precise, operational.
PHILOSOPHY

设计哲学

后台不是展示空间，而是决策空间。这个风格的目标不是“空”，而是把视觉噪音压到最低， 让用户在列表、状态、筛选和操作之间快速建立判断。

  1. 少即是可控 默认只暴露必要信息，更多动作以轻按钮、弹层或详情区承接。
  2. 线条就是结构 使用 1px 分割线和稳定栅格组织页面，减少容器装饰。
  3. 颜色只表达状态 黑白灰承担层级，低饱和绿、铜、蓝灰只用于成功、风险、等待。
  4. 动效服务反馈 交互动效短、轻、可忽略，不抢走业务信息的注意力。
  5. 组件先为扫描而生 标题、数值、标签和操作区有固定节奏，让复杂页面也能被快速扫读。
